# Palette Interactive Pane Design

> Date: 2026-04-06
> Status: Approved (POC validated)
> Depends on: `2026-04-06-zellij-plugin-design.md`

## Problem

The command palette tried to replicate CLI interactive flows (server selection menus, conditional prompts, proxy input) inside a WASM plugin. This approach had fundamental issues:

1. **Input limitations** — floating pane width truncated long inputs (e.g. proxy `ip:port`)
2. **Inconsistent prompts** — palette showed generic `(optional, Enter to skip)` instead of CLI's rich descriptions
3. **Missing conditional logic** — CLI's `project create` has branching prompts (server choice triggers port/tls/password; claude type triggers proxy), impossible to replicate without duplicating business logic
4. **Maintenance burden** — every CLI change required parallel plugin updates

## Solution: Palette as Launcher, CLI as Executor

The palette's role is **command discovery + launch**, not interactive form filling.

```
User presses Ctrl-K
    → Palette: fuzzy search commands
    → Palette: collect required positional args only (e.g. name)
    → Palette: open_command_pane_floating(zchat <cmd> <args>)
    → Palette: hide_self()
    → User: interacts with real CLI in floating terminal pane
```

### What Changed

| Before | After |
|--------|-------|
| `run_command` (background, no TTY) | `open_command_pane_floating` (interactive terminal) |
| Plugin collects ALL args (required + optional) | Plugin collects only required positional args |
| Plugin renders Result/Executing states | CLI output visible directly in terminal pane |
| Plugin needs RunCommandResult handler | No result handling needed |
| Plugin needs choices/labels for optional args | CLI handles its own interactive menus |

### Plugin State Machine (simplified)

```
CommandFilter → [select command]
    → ArgInput (for each required positional arg)
        → [Enter] → execute_command()
            → open_command_pane_floating(...)
            → hide_self()
            → reset to CommandFilter
```

States removed: `Executing`, `Result`, `ArgSelect` (for optional args).

`ArgSelect` is retained only for required args with runtime sources (e.g. `agent stop` needs to select from running agents via TabUpdate).

## Key API: `open_command_pane_floating`

```rust
fn execute_command(&mut self) {
    let cmd = commands::build_cli_args(
        &self.zchat_bin,
        &self.project_name,
        &self.current_command,
        &self.collected_args,
    );
    let command = CommandToRun {
        path: PathBuf::from(&cmd[0]),
        args: cmd[1..].to_vec(),
        cwd: None,
    };
    open_command_pane_floating(command, None, BTreeMap::new());
    hide_self();
    self.state = PaletteState::default();
}
```

- Requires `PermissionType::RunCommands` (already granted)
- No sandbox restrictions — Zellij manages the pane lifecycle
- Terminal pane has full TTY — readline, paste, color, all work
- Pane closes when command exits (user sees output, presses Enter or closes)

## What the Plugin Still Does

1. **Command discovery** — reads `commands_json` from KDL config (pre-generated by CLI)
2. **Fuzzy matching** — filters command list as user types
3. **Required arg collection** — only for positional args that CLI can't prompt for (e.g. `name` in `project create`, `agent create`)
4. **Runtime arg selection** — `agent stop/focus/restart` shows running agents from TabUpdate; `project use` shows sessions from SessionUpdate
5. **Launch** — opens floating terminal pane with the assembled command

## What the Plugin Does NOT Do

- No optional arg collection (CLI prompts handle this)
- No conditional branching logic
- No result parsing or display
- No input validation
- No choices/labels for static config (servers, templates)

## Commands JSON (simplified)

The `choices` field in `list-commands` output is no longer consumed by the palette for optional args. It remains available for other integrations (shell completion, WeeChat plugin) but the palette ignores it.

The palette only uses:
- `name` — for fuzzy matching
- `args[].name` — for positional arg prompts
- `args[].required` — to filter which args to collect
- `args[].source` — to show runtime selection lists (running_agents, projects)

## Acceptance Criteria

1. `Ctrl-K` → type "ag cr" → select "agent create" → type name → floating terminal opens with `zchat agent create <name>` — **validated**
2. `Ctrl-K` → "pr cr" → type name → floating terminal shows full server selection menu identical to `zchat project create` — **validated**
3. `Ctrl-K` → "shutdown" → no args needed → floating terminal runs `zchat shutdown` immediately — to validate
4. `Ctrl-K` → "ag stop" → shows running agents list from TabUpdate → select → floating terminal runs `zchat agent stop <name>` — to validate
5. Proxy input works with arbitrary length (full terminal, no truncation) — **validated** (terminal pane has no width limit on input)
6. Plugin code is <200 lines (main.rs) — **validated** (reduced from ~460 to ~390)
