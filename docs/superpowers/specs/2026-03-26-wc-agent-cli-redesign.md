# wc-agent CLI Redesign — Project System + Two-Level Subcommands

**Date**: 2026-03-26
**Scope**: Restructure wc-agent CLI with multi-project support, Typer-based `project`/`irc`/`agent` subgroups, interactive config generation, manual test guide.

## Overview

Redesign the wc-agent CLI to support a clean manual testing workflow. Key changes:
- **Project system**: `~/.wc-agent/projects/<name>/` — each project is an independent IRC environment with its own config and agent state
- **Two-level subcommands**: `project`, `irc`, `agent` groups using Typer
- **Interactive config**: `project create` prompts for IRC server/nick/channels, generates config
- **No static config in project root**: `weechat-claude.toml` is replaced by per-project config in `~/.wc-agent/`

## Command Structure

```
# Project management
wc-agent project create <name>         # Interactive setup: IRC server, nick, channels
wc-agent project list                  # List all projects
wc-agent project use <name>            # Set default project
wc-agent project remove <name>         # Remove project config + state
wc-agent project show [name]           # Show project config

# IRC infrastructure
wc-agent irc daemon start              # Start ergo IRC server
wc-agent irc daemon stop               # Stop ergo
wc-agent irc start [--nick <n>]        # Start WeeChat in tmux + auto-connect IRC
wc-agent irc stop                      # Stop WeeChat (/quit)
wc-agent irc status                    # Show IRC server + client status

# Agent lifecycle
wc-agent agent create <name>           # Create agent (claude + channel-server)
  [--workspace <path>] [--channels <c>]
wc-agent agent stop <name>             # Stop agent
wc-agent agent list                    # List all agents with status
wc-agent agent status <name>           # Single agent details
wc-agent agent send <name> <text>      # Send text to agent's tmux pane
wc-agent agent restart <name>          # Restart agent

# Convenience
wc-agent shutdown                      # Stop all agents + WeeChat + ergo

# Global option (all commands)
wc-agent --project <name> <cmd>        # Specify project (overrides auto-detection)
```

## Project System

### Directory Structure

```
~/.wc-agent/
  default                              # Contains default project name (set by `project use`)
  projects/
    my-team/
      config.toml                      # Generated by `project create`
      state.json                       # Runtime state (agents, panes)
    local-dev/
      config.toml
      state.json
```

### Project Resolution Order

1. `--project my-team` — explicit CLI argument
2. `.wc-agent` file in current directory or parent directories — contains project name (like `.git`)
3. `~/.wc-agent/default` file — set by `wc-agent project use`
4. None found → error: `No project selected. Run 'wc-agent project create <name>' or use '--project <name>'.`

### Generated config.toml

```toml

server = "irc.libera.chat"
port = 6697
tls = true
password = ""

[agents]
default_channels = ["#my-team", "#general"]
username = "alice"
```

### CLI Parameter Priority

For parameters that exist in both config and CLI:
1. CLI argument (e.g., `--nick bob`) — highest priority, does not modify config
2. Config value (e.g., `username = "alice"`) — default
3. System fallback (e.g., `$USER`) — if config value is empty

## Pane Lifecycle

All tmux panes (WeeChat, agents) are managed passively:

- **Create**: `irc start` / `agent create` spawn panes, record `pane_id` in `state.json`
- **Detect**: `agent list/status/send` check if pane exists (`tmux list-panes`)
  - If pane gone → mark `offline` in state, report to user
- **Recover**: User must manually `agent create` again — no auto-restart
- **Cleanup**: `agent stop` sends `/exit` to pane, cleans workspace, updates state. If pane already gone, just cleans state.

## Deletion of `weechat-claude.toml`

The project-root `weechat-claude.toml` is no longer needed. Config lives in `~/.wc-agent/projects/<name>/config.toml`. The old file will be deleted from the repo.


This eliminates: CLI path resolution, subprocess overhead, Typer dependency in channel-server.

### `start.sh`

**Current**: `python3 wc-agent/cli.py --config $CONFIG start --workspace $WORKSPACE`
**Change**: Three separate commands with project context:
```bash
wc-agent --project local irc daemon start
wc-agent --project local irc start
wc-agent --project local agent create agent0 --workspace "$WORKSPACE"
```

Or simpler if default project is set: `wc-agent irc daemon start && wc-agent irc start && wc-agent agent create agent0`

### `stop.sh`

**Current**: `python3 wc-agent/cli.py --config $CONFIG shutdown`
**Change**: `wc-agent shutdown` (uses project resolution to find config).

### `weechat-claude.toml`

**Delete** from project root. Config now lives in `~/.wc-agent/projects/<name>/config.toml`.

### shutdown command

Typer doesn't support mixing `@app.command()` with `app.add_typer()` on the same app. Solution: `shutdown` is a subgroup with `invoke_without_command=True`:

```python
# shutdown as a subgroup that invokes immediately
@app.command("shutdown")
# Actually, use Click's underlying group mechanism:
shutdown_app = typer.Typer(invoke_without_command=True)
app.add_typer(shutdown_app, name="shutdown")

@shutdown_app.callback(invoke_without_command=True)
def shutdown():
    """Stop all agents + WeeChat + ergo."""

```

### Global options

```python
@app.callback()
def main(
    ctx: typer.Context,
    project: str = typer.Option(None, help="Project name (overrides auto-detection)"),
    tmux_session: str = typer.Option("weechat-claude", help="tmux session name"),
):
    """Claude Code agent lifecycle management."""
    ctx.ensure_object(dict)
    # Resolve project: --project > .wc-agent file > ~/.wc-agent/default
    resolved = project or resolve_project_from_cwd() or resolve_default_project()
    if resolved and ctx.invoked_subcommand not in ("project",):
        # Load project config
        ctx.obj["project"] = resolved
        ctx.obj["config"] = load_project_config(resolved)
    ctx.obj["tmux_session"] = tmux_session


def resolve_project_from_cwd() -> str | None:
    """Walk up from cwd looking for .wc-agent file containing project name."""
    path = os.getcwd()
    while path != "/":
        marker = os.path.join(path, ".wc-agent")
        if os.path.isfile(marker):
            return open(marker).read().strip()
        path = os.path.dirname(path)
    return None


def resolve_default_project() -> str | None:
    """Read ~/.wc-agent/default."""
    default_file = os.path.join(WC_AGENT_DIR, "default")
    if os.path.isfile(default_file):
        return open(default_file).read().strip()
    return None
```

## Idempotency

All start commands are idempotent:
- `irc daemon start`: if ergo already running (`pgrep -x ergo`), print status and return
- `irc start`: if WeeChat pane already exists and alive, print status and return
- `agent create <name>`: if agent already running, error

All stop commands are idempotent:
- `irc daemon stop`: if ergo not running, print "not running" and return
- `irc stop`: if WeeChat pane not found, print "not running" and return
- `agent stop <name>`: if agent already offline, error

## Agent Name Scoping in server.py

When `server.py` directly imports `AgentManager`, it must pass the already-scoped name to avoid double-scoping:

```python
# In server.py _handle_create_agent:
from wc_agent.agent_manager import AgentManager

# Extract username from parent agent name
username = AGENT_NAME.split("-")[0]  # alice-agent0 → alice
scoped = scoped_name(name, username)  # agent2 → alice-agent2

# Create manager and call create() with scoped name
# AgentManager.create() will see "-" in name and skip re-scoping
mgr = AgentManager(...)
mgr.create(scoped, ...)
```

## Manual Test Guide

`docs/e2e-manual-test.md`:

```markdown
# E2E Manual Test Guide

## Prerequisites

- ergo installed: `~/.local/bin/ergo`
- ergo languages: `~/.local/share/ergo/languages/`
- tmux installed
- wc-agent installed (or run via `python3 wc-agent/cli.py`)

## Step-by-Step

### 0. Create Project (first time only)

    $ wc-agent project create local-test
    Creating project 'local-test':
      IRC server [127.0.0.1]:
      IRC port [6667]:
      TLS [false]:
      Password []:
      Nickname [alice]: alice
      Default channels [#general]:
    Project 'local-test' created.

    $ wc-agent project use local-test

### 1. Start IRC Server

    $ wc-agent irc daemon start
    Starting ergo IRC server...
    ergo running (pid 12345).

### 2. Start WeeChat

    $ wc-agent irc start
    Starting WeeChat in tmux...
    WeeChat started in pane %5.

Attach to observe (iTerm2):

    $ tmux -CC attach -t weechat-claude

Or standard tmux:

    $ tmux attach -t weechat-claude

### 3. Verify IRC Status

    $ wc-agent irc status

Should show both server and client running.

### 4. Create Primary Agent

    $ wc-agent agent create agent0

In WeeChat, you should see `alice-agent0` join #general.

### 5. Test Agent Reply

    $ wc-agent agent send agent0 'Use the reply MCP tool to send "Hello everyone, agent0 here!" to #general'

Watch WeeChat #general for the message.

### 6. Test @mention

In WeeChat #general, type:

    @alice-agent0 what is the capital of France?

Agent should auto-respond in the channel.

### 7. Check Agent Status

    $ wc-agent agent list
    $ wc-agent agent status agent0

### 8. Create Second Agent

    $ wc-agent agent create helper
    $ wc-agent agent list

Note: `agent create helper` produces `alice-helper` on IRC (username prefix from config).

### 9. Agent-to-Agent Communication

    $ wc-agent agent send agent0 'Use the reply tool to send "hello helper, please respond with PONG" to "alice-helper"'

### 10. Stop Agents

    $ wc-agent agent stop helper
    $ wc-agent agent list

### 11. Shutdown Everything

    $ wc-agent shutdown

Verify: no ergo process, no tmux session, no agent panes.
```
