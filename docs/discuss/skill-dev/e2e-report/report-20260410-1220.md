## 发现的问题

### BUG-1: verify-skill-output.sh mode 字段 regex 不兼容无引号 YAML — **已修复**

**严重程度**: P2 (低)
**位置**: `docs/discuss/skill-dev/verify-skill-output.sh:258`
**状态**: ✅ 已修复并验证 (8/8 通过)

### BUG-2: init-artifact-space.sh 未检查 git 仓库 — **已修复**

**严重程度**: P3 (建议)
**位置**: `~/.claude/skills/dev-loop-skills/skills/skill-6-artifact-registry/scripts/init-artifact-space.sh`
**状态**: ✅ 已修复，symlink 自动生效

### BUG-3: `claude -p` 默认权限模式阻止 Skill 工具调用 — **已修复**

**严重程度**: P0 (关键)
**现象**: 第一轮交互测试中，`claude -p` 默认权限拒绝 Bash 工具调用，Skill 6 输出 "看起来这个命令需要你的批准才能运行"，Skill 1 输出 "测试命令被反复拒绝"
**修复**: `run-skill-e2e.py` 添加 `--permission-mode bypassPermissions` 参数
**影响**: 第二轮测试通过率从 1/7 提升至 6/7

### TIMEOUT-1: Skill 5 simulate 模式超过 300s

**严重程度**: P2 (需调优)
**现象**: Skill 5 simulate 模式在 300s 内未完成，但实际已在磁盘上写入了 eval-doc (9730 bytes) 和 registry 注册
**原因**: simulate 模式需要读取大量代码、模拟多个 testcase、写入文件并注册 — 总工作量超过 5 分钟
**证据**: `eval-agent-dm-001.md` 在 15:01 创建，但超时在 15:01 后触发（进程被 kill 时已完成文件写入但可能未完成输出返回）
**建议**: 将 Skill 5 timeout 提高到 600s，或将 simulate 拆分为"生成 testcase"和"写入 artifact"两步

### NOTE-1: Python 版本差异

**严重程度**: 信息
**现象**: `uv run python3 --version` 报告 3.13.5，但 env-check.sh 报告 3.14.3
**原因**: uv 虚拟环境可能配置了不同的 Python 解释器
**影响**: 无功能影响，测试计划中要求 >= 3.14 可能需要更新

### NOTE-2: 单元测试基线有 1 个已知失败

**现象**: `test_irc_check.py::test_unreachable_server_raises` 在 ergo 运行时失败
**原因**: 测试假设端口不可达，但本地 ergo 正在监听
**影响**: 不影响 skill E2E 测试

## 交互式测试方法

使用 `run-skill-e2e.py`，基于 skill-creator 的 `claude -p` (pipe mode) 方法自动化执行 skill：

```bash
# 全部测试
uv run python3 docs/discuss/skill-dev/run-skill-e2e.py --all

# 单个 skill
uv run python3 docs/discuss/skill-dev/run-skill-e2e.py --skill 6

# dry-run
uv run python3 docs/discuss/skill-dev/run-skill-e2e.py --all --dry-run
```

**关键参数**: `--permission-mode bypassPermissions` 确保 Skill 可以执行 Bash、Write 等工具。
